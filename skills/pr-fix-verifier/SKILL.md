# PR Fix Verifier Skill

This skill outlines the strict workflow for writing, executing, and reporting validation tests to prove that a specific codebase issue/task has been resolved. 

Deterministic test coverage is the absolute gatekeeper for merging. No code change shall be merged by automated systems unless it passes its corresponding verification test suite.

---

## Core Objectives
1. **Zero Silent Failures:** Ensure that regressions or logical errors are caught before merging into the main branch.
2. **Deterministic Proof:** Every task in `TASKLIST.md` or `./issues/` must be mapped to a test case that fails when the issue is present and passes once the issue is fixed.
3. **Execution Guard:** Provide a clear API and directory structure that `ci_reviewer.py` can programmatically run to evaluate Pull Request viability.

---

## 1. Directory Structure and Naming Conventions

All verification tests generated by agents must live in a dedicated `tests/` directory at the repository root.

```text
repository-root/
├── issues/
│   └── task_1_proxy_py_ANSI_ESCAPE_bug.md
├── tests/
│   ├── __init__.py
│   └── test_task_1.py  <-- Explicit mapping to task ID
├── proxy.py
└── ci_reviewer.py
